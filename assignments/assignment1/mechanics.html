Twitch Delay
One issue with reading the IRC chatroom adjoining the Twitch stream is that the Twitch video stream is always about 10 seconds behind the IRC chatroom, meaning that when a viewer types a command in the IRC chatroom, the game recieves it immediately, but the viewer won't see the action on screen for 10 seconds. This problem was known of before development and the game was built around this constraint. As such, nearly all actions are events that take place over time such that their effect in a given moment is less important then their effect over time, giving the viewer time to respond even with a 10 second delay. The streamer has no delay and is given the option to override many crew choices so that if the crew are not responding fast enough or at all, the Captain can still perform actions.
Twitch IRC Authentication
Because viewers in the IRC chatrooms are authenticated through the Twitch API, we don't need to authenticate again besides using their username as the key to their character's save file. From here, we can save and load their character's information to a save file using their username.
Unreal Component System
The spaceship is meant to be a complicated system of rooms and ship systems, requiring multiple people to fully utilize. In order to maintain this in a reasonable means through programming, the spaceship's blueprint in the Unreal Engine is made up of multiple sub-components, each of which is saved and loaded independently. The components require certain rooms to be operational in order for that component to be operational (for example, the Impulse Component, which allows ship movement, requires that the Engines, Core and Bridge be operational. Each of the different components can communicate with each other through the ship's base blueprint that they're all attached to as components.
Unreal Data Binding
One benefit of using Unreal Engine is it's superb user interface development system, UMG. This allows us to bind the data values of spaceship components, crew and rooms to UI elements, keeping them up to date in realtime without complicated spaghetti code. This is important because showing lots of information onscreen simultaneously is a key to this design method.
  • Networking-Light: By only reading from an IRC chatroom, the game itself only uses a minimal amount of bandwidth.
  • Cooperative Gameplay: By utilizing Twitch's chatroom system, we can create a unique cooperative game over the internet.
  • Local Persistence: Because the viewers communicate with the game remotely, all the save data can be held on the streamer's local machine.
